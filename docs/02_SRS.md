## 2. 系統概述

### 2.1 目的

打造一套高勝率、多市場、全自動化的量化交易決策系統，融合四套既有系統精華，覆蓋台股與美股市場，從選股到出場全流程紀律化執行。

### 2.2 範疇

| 市場               | 優先級    | 說明             |
| ------------------ | --------- | ---------------- |
| 台股 (TWSE/TPEx)   | P0 核心   | 所有功能優先支援 |
| 美股 (NYSE/NASDAQ) | P1 第二期 | Phase 7 擴展     |
| ~~中國 A 股~~      | 已排除    | 不納入系統範疇   |

### 2.3 目標使用者

單一使用者（系統擁有者），具備中高階投資知識，熟悉技術分析、籌碼分析、基本面分析。

### 2.4 核心設計決策

| 決策          | 內容                                                                                |
| ------------- | ----------------------------------------------------------------------------------- |
| 選股主邏輯    | 四大主軸（產業輪動→題材催化→資金流向→優於大盤RS）+ 三大面向（技術面+基本面+籌碼面） |
| 篩選規則      | 三面向至少兩面向同時為正才納入最終清單                                              |
| 七流派/ML/SMC | 降為輔助確認，非選股主邏輯                                                          |
| 資料庫        | PostgreSQL（淘汰 SQLite/JSON），熱資料 Redis                                        |
| 即時性        | 三層+ Fallback，自動重連，斷線告警                                                  |

### 2.5 優先級定義

| 優先級 | 對應階段                         | 說明     |
| ------ | -------------------------------- | -------- |
| P0     | Phase 1–2（基礎建設 + 核心引擎） | 必須實作 |
| P1     | Phase 3–4（即時系統 + 回測風控） | 應該實作 |
| P2     | Phase 5+（自動化 + UI + 美股）   | 可以實作 |

### FR-SEL: 選股邏輯

#### FR-SEL-01 四大主軸評分引擎

| 欄位         | 內容                                                                                                                                                                                                      |
| ------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **ID**       | FR-SEL-01                                                                                                                                                                                                 |
| **名稱**     | 四大主軸評分引擎                                                                                                                                                                                          |
| **描述**     | 系統須依序評估「產業輪動強度」「題材催化事件」「資金流向方向」「個股相對強弱 RS 優於大盤」四個主軸，各主軸產生標準化分數（0–100），並以加權方式合成「主軸總分」。權重可由使用者於設定介面調整，預設等權。 |
| **優先級**   | P0                                                                                                                                                                                                        |
| **來源系統** | Signal Radar（七流派五層訊號）、Atlas（四大模組融合）、MasterTalks（資金五維評分）                                                                                                                        |
| **驗收條件** | 1. 輸入任一台股標的，系統於 5 秒內回傳四主軸各自分數與加權總分 2. 修改權重後重新計算結果與手動驗算一致（誤差 < 0.01） 3. 無主軸資料缺失時標註「N/A」而非報錯                                              |

#### FR-SEL-02 三大面向交叉篩選

| 欄位         | 內容                                                                                                                                                                                                                                                    |
| ------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **ID**       | FR-SEL-02                                                                                                                                                                                                                                               |
| **名稱**     | 三大面向交叉篩選（技術 / 基本 / 籌碼）                                                                                                                                                                                                                  |
| **描述**     | 對每檔標的分別計算技術面信號（均線排列、動能指標、型態辨識）、基本面信號（營收年增率、EPS、本益比河流位置）、籌碼面信號（三大法人買賣超、融資融券變化、集中度），各面向輸出「正 / 中性 / 負」三態。**硬性規則：至少兩面向同時為「正」才納入候選清單**。 |
| **優先級**   | P0                                                                                                                                                                                                                                                      |
| **來源系統** | 全部四套                                                                                                                                                                                                                                                |
| **驗收條件** | 1. 給定 10 檔測試標的，系統輸出三面向各自判定結果與最終「納入 / 排除」決策 2. 僅一面向為正的標的確實被排除 3. 三面向判定邏輯可透過 config 檔調整閾值，修改後不需重啟即生效                                                                              |

#### FR-SEL-03 選股收斂流程

| 欄位         | 內容                                                                                                                                                                      |
| ------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **ID**       | FR-SEL-03                                                                                                                                                                 |
| **名稱**     | 從全市場 ~2000 檔收斂至 ~50 檔候選                                                                                                                                        |
| **描述**     | 整合 FR-SEL-01 與 FR-SEL-02，按「主軸總分」降冪排序，取前 N 檔（預設 50，可調）。收斂過程須記錄每檔被排除的原因（哪個面向不通過、主軸分數不足等），供使用者查閱篩選紀錄。 |
| **優先級**   | P0                                                                                                                                                                        |
| **來源系統** | 新設計（整合層）                                                                                                                                                          |
| **驗收條件** | 1. 全台股掃描（~2000 檔）於 60 秒內完成 2. 輸出候選清單筆數可調且實際筆數符合設定 3. 篩選紀錄可匯出 CSV，包含每檔的四主軸分數、三面向判定、排除原因                       |

#### FR-SEL-04 輔助確認層（七流派 / ML / SMC）

| 欄位         | 內容                                                                                                                                                                                                                                      |
| ------------ | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **ID**       | FR-SEL-04                                                                                                                                                                                                                                 |
| **名稱**     | 七流派 / ML / SMC 輔助確認                                                                                                                                                                                                                |
| **描述**     | 對 FR-SEL-03 產出的 ~50 檔候選，額外執行：(a) 七流派共識度評分 (b) RandomForest ML 預測（防未來函數機制）(c) SMC 結構確認（OB / FVG / Liquidity / CRT）。結果以「輔助信心度」呈現（高 / 中 / 低），**不影響主邏輯篩選結果**，僅作為標註。 |
| **優先級**   | P0                                                                                                                                                                                                                                        |
| **來源系統** | Signal Radar（七流派 + SMC）、MasterTalks（ML RandomForest）                                                                                                                                                                              |
| **驗收條件** | 1. 輔助信心度欄位出現在候選清單中但不改變排序 2. ML 預測通過防未來函數測試（使用 T-1 資料預測 T 日方向） 3. 關閉輔助確認層後，主邏輯候選清單結果完全不變                                                                                  |

### FR-UNI: 選股池管理
