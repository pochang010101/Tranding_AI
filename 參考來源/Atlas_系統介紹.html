1. 系統簡介與定位

Atlas Signal Radar 是一套台股量價研究與訊號分析系統，核心定位是「盤後選股 + 盤中即時觸發推送」的決策助手，服務當沖、隔日沖與短線交易，而非一套研究平台。它把訊號送到你眼前、把進場時機推播到手機，最後的下單與紀律由人來執行。

系統融合三大量價流派，交叉印證以提高訊號品質：

流派內容
① 經典量價派量價時角 ＋ 供需原理
② 籌碼學派極短線 ／ 籌碼學派
③ 學院派學院派量價 ＋ 市面常見短線策略
底層技術組成：Streamlit 儀表板 ＋ GitHub Actions 自動掃描 ＋ 群益即時報價。GitHub：pochang010101/atlas-signal-radar（private）。

2. 三層部署架構

Atlas 把職責切成三個獨立運行的層，各自在最適合的環境執行，彼此透過資料檔與推播串接：

層環境負責
Streamlit 儀表板Streamlit Cloud ／ 本機UI、即時掃描、回測、K 線檢視
排程掃描 + 推播GitHub Actions（daily-scan workflow）盤後自動掃描 → commit 選股資料 → Discord／LINE
群益即時行情本機 Windows onlyCapital SKCOM API，雲端自動降級至 TWSE MIS

這樣的切分讓「自動化掃描＋推播」可以脫離本機、由 GitHub Actions 無人值守跑；而需要 Windows 原生 COM 元件的群益即時行情則留在本機，雲端環境自動降級為跨平台的 TWSE MIS，不會因環境差異而停擺。

整合契約層（importlib 的關鍵設計）

核心關鍵在 signal_radar/integration.py：它用 importlib 動態載入中文檔名的傳統引擎，對外提供英文命名的穩定介面（如 run_postmarket_scan、load_ohlcv、get_market_regime）。任何引擎 import 失敗，一律以 try/except 優雅降級 — 即使某支引擎掛掉，Streamlit 也不會整個崩潰，只會少掉對應功能。
