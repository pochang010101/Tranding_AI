Atlas Signal Radar 使用者操作手冊

台股量價研究與訊號分析系統 — 從啟動、盤前看盤、盤後選股到盤中即時觸發的完整操作指南。

1. 快速開始

Atlas 是一套以「盤後選股 + 盤中即時觸發」為核心的當沖／隔日沖／短線決策助手。前端為 Streamlit 儀表板，後端為 Python 引擎與 CLI 工具。

安裝

pip install -r requirements.txt
# Streamlit 儀表板額外需要：
pip install -r requirements_streamlit.txt

啟動儀表板

python run_streamlit.py        # 啟動 Streamlit 儀表板（port 8501）

啟動後瀏覽器開啟 http://localhost:8501，左側側邊欄即可在九個分頁間切換。

主題切換：儀表板支援深／亮雙主題，並依台股慣例採「紅漲綠跌」配色（賺＝紅、賠＝綠）。任何主題下文字都應清晰可讀。
即時報價來源：本機 Windows 環境走群益 SKCOM 取得即時價；雲端或非 Windows 環境會自動降級為 TWSE MIS。兩者對使用者而言介面一致，無須手動切換。

2. 每日操作流程

一天的標準使用節奏：盤前看市場環境 → 盤後選股建立明日候選 → 盤中即時雷達盯訊號 → 收盤後記錄交易日誌。

① 盤前：看「市場環境因子」

開盤前先到 1 市場環境因子 頁看一眼綜合閘門結論——今天是否放行開新倉。同時掃過大盤多空、國際面（美股隔夜／費半／台積電 ADR）、期現基差與個股事件，決定今天的操作積極度。

② 盤後：跑「盤後選股」

收盤後到 3 盤後選股 頁一鍵掃描候選池，產出買進候選／賣出警示／補漲候補三張清單，作為明日操作的目標股。

建議時間：價量為盤後 EOD（每日終值），約收盤（13:30）後 1 小時就緒；建議 14:30 後再看盤後選股，資料較完整。

③ 盤中：盯「盤中即時雷達」

開盤後到 5 盤中即時雷達 頁，輸入／載入觀察清單，盯即時報價與盤中六訊號（多 B1/B2/B3、空 S1/S2/S3）的即時觸發。觸發後依個人紀律到群益 App 手動下單。

④ 收盤後：記「交易日誌」

當日操作結束後到 6 交易日誌與績效 頁記單，檢視累積損益、月報酬與風控冷靜期狀態，形成可回顧的紀錄。

3. 九大分頁逐頁說明

1 市場環境因子

用途：開盤前的「敢不敢開新倉」總判斷頁。

看什麼：頁面最上方為綜合閘門結論（今日是否放行開新倉）；接著是大盤 TWII、VIX、費半 SOX、台積電 ADR；美股隔夜（道瓊／那斯達克／標普 500）與台指期基差；以及個股事件（增減資／可轉債 CB）。頁首右上角顯示「📅 資料日期」，提醒你看的是哪一天的數據。

常見操作：頁首有「🔄 強制刷新因子」按鈕可重抓全部因子；當某區塊顯示 N/A 時，該區會出現「🔄 重新強制抓取」按鈕，按下會繞過快取重抓該批資料（詳見第 4 節）。

2 RS 強度與產業輪動

用途：只買最強族群中的最強股——找出相對強度（RS）領先的個股與輪動中的產業。

看什麼：個股 RS 強度排行（IBD 式 1–99 百分位）、產業輪動（各產業相對大盤強弱）與補漲候補、進階市場因子（外資台指期淨額／融資維持率／大盤型態）。

常見操作：用 RS 排行縮小選股範圍，再對照產業輪動確認族群方向。資料未就緒時會優雅降級顯示，不報錯。

3 盤後選股

用途：盤後一鍵掃描候選池，產出隔日操作清單。

看什麼：買進候選、賣出警示、補漲候補三張清單，表格可排序／篩選。

常見操作：可在「掃描設定」自訂掃描清單（逗號分隔，留空＝引擎預設 candidates.json）、選資料區間與回看天數，再一鍵掃描；點選個股可跳轉至 K 線檢視。

籌碼提示：若籌碼資料尚未抓取，頁面會提示「請點按鈕抓取，或執行 python 籌碼面抓取.py」。三大法人／融資券由證交所約 15:00–16:00 後才公布當日，太早跑可能抓不到當日。

4 K 線量價檢視

用途：核心看圖頁，逐檔檢視技術 K 線與成交量。

看什麼：K 線 + 成交量雙圖（含 MA8／21／55、量能均線 VMA5／13／34、可選布林通道與 MACD 副圖），並可疊加策略訊號的買賣三角。

常見操作：直接輸入代號查圖，或由「盤後選股」頁點股票跳轉而來。

5 盤中即時雷達

用途：盤中即時報價 + 盤中六訊號即時觸發。

看什麼：觀察股的即時報價卡，以及盤中六訊號觸發清單——多訊號 B1／B2／B3（紅）、空訊號 S1／S2／S3（綠）。

常見操作：手動點刷新更新報價與訊號（不做自動輪詢迴圈，以免卡住畫面）。報價來源本機走群益 SKCOM、雲端／非 Windows 自動降級 TWSE MIS。

6 交易日誌與績效

用途：記錄交易、追蹤績效與風控狀態。

看什麼：績效總覽（KPI 卡 + 累積損益曲線 + 月報酬表）、交易紀錄表格、風控冷靜期狀態、模擬倉持倉。

常見操作：以表單記單；損益依台股慣例著色（賺＝紅、賠＝綠）。

7 策略熱力圖與深度報告

用途：進階分析——把月度績效、策略熱力圖與深度報告整進前端。

看什麼：月度績效（月勝率／獲利因子 PF／最大回撤）、各策略表現熱力圖與個股深度報告。

常見操作：用熱力圖快速比較哪些策略近期有效，再切深度報告看單檔細節。

8 回測與基因優化

用途：單檔 × 策略回測，以及策略參數的基因演算法優化。

看什麼：回測正規化績效（勝率％／獲利因子／總報酬％／交易次數）；基因優化的最佳基因、Top-N 基因表與演化績效曲線。

常見操作：選股票與策略跑回測；最佳基因可一鍵回填到側邊欄 config 覆寫，套用於後續分析。

9 策略與參數

用途：策略與參數「圖鑑頁」，一眼看懂系統所有策略與可調參數。

看什麼：策略總表（多單／空單／全部分頁）、三流派融合模組與理念金句、可調參數說明（含目前值）。

常見操作：查每個門檻屬於哪個流派、有什麼意義、預設／目前值是多少——作為理解前述各頁結果的對照表。

4. 強制刷新／重新強制抓取按鈕怎麼用

這兩個按鈕都在 1 市場環境因子 頁，用來處理「資料看起來不對或抓不到」的情況。

為什麼會出現 N/A

因子資料來自多個外部免費來源。當某來源暫時無回應、超出公布時點，或當下被快取的是舊的空值時，該因子會以 N/A 顯示（系統設計為降級不報錯，不會整頁掛掉）。

兩個按鈕的差別

按鈕位置作用何時該按
🔄 強制刷新因子頁首重抓全部因子（繞過快取）整頁數據看起來是舊的、或想拿最新一輪數值時
🔄 重新強制抓取出現在 N/A 區塊旁只繞過快取重抓「該批」資料單一區塊顯示 N/A、懷疑是暫時抓取失敗時（新功能）
提醒：若是因為「還沒到公布時點」造成的 N/A（例如太早看當日籌碼），重按也抓不到——這時請依第 5 節的建議時間稍後再看，而非反覆強制重抓。

5. 資料更新時點與建議操作時間

不同資料的就緒時間不同，太早看會抓不到當日值。下表為操作時點建議：

資料類型來源／性質就緒時點建議操作時間
價量（K 線、成交量）盤後 EOD 每日終值收盤（13:30）後約 1 小時14:30 後再看盤後選股
三大法人買賣超證交所公布約 15:00–16:00 後16:00 後抓籌碼較穩
融資融券證交所公布約 15:00–16:00 後16:00 後抓籌碼較穩
市場環境因子（國際面）多個免費來源美股隔夜收盤後盤前查看；N/A 可按重新強制抓取
盤中即時報價群益 SKCOM／TWSE MIS盤中即時開盤後手動刷新
自動回溯：若太早抓籌碼導致當日尚未公布，系統會自動回溯到最近一個有公布的交易日，不會讓你拿到空白資料。

7. 常見問題

Q1. 因子顯示 N/A 怎麼辦？

先確認是否「還沒到公布時點」（見第 5 節）。若已過就緒時間仍 N/A，按該區塊旁的「🔄 重新強制抓取」繞過快取重抓；若整頁都舊，按頁首「🔄 強制刷新因子」。

Q2. 籌碼資料抓不到當日？

三大法人與融資券由證交所約 15:00–16:00 後才公布當日，太早抓會抓不到。請在 16:00 後再抓，或執行 python 籌碼面抓取.py。系統會自動回溯到最近有公布的交易日，所以不會空白。

Q3. 盤後選股提示要抓籌碼？

表示籌碼資料尚未就緒。依頁面提示點按鈕抓取，或在命令列執行 python 籌碼面抓取.py 後重新整理頁面。

Q4. 改了程式但儀表板沒變化？

修改 streamlit_app/ 下的檔案後需重啟 Streamlit 才生效（模組有快取）。停掉再重跑 python run_streamlit.py 即可。

Q5. 盤中報價沒有即時更新？

盤中即時雷達不做自動輪詢，需手動點刷新。另外本機 Windows 走群益 SKCOM，雲端／非 Windows 自動降級 TWSE MIS——確認所在環境是否符合預期來源。
